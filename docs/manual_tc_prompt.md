# 매뉴얼 TC 생성 프롬프트

ChatGPT, Claude 등 AI에 아래 프롬프트를 붙여넣고, 하단 [티켓 정보] 부분만 교체하여 사용하세요.

---

## 프롬프트 (복사해서 사용)

```
당신은 경력 있는 QA 엔지니어입니다.
아래 티켓 정보를 바탕으로 매뉴얼 테스트 케이스를 작성해주세요.

작성 기준:
- 정상 흐름(Happy Path): 기능이 의도대로 동작하는 케이스
- 예외 처리: 잘못된 입력, 권한 없음, 네트워크 오류 등
- 경계값: 최소/최대 입력값, 빈 값, 특수문자 등

우선순위 기준:
- High: 핵심 기능, 서비스 불가 수준의 장애 가능성
- Medium: 주요 기능이지만 우회 가능
- Low: 부가 기능, UI/UX 개선 수준

반드시 아래 JSON 배열 형식으로만 응답하세요.
설명, 마크다운 코드블록 없이 JSON만 출력하세요.

[
  {
    "tc_id": "TC-001",
    "테스트항목": "무엇을 테스트하는지 한 줄 요약",
    "사전조건": "테스트 실행 전 준비 상태 (예: 로그인된 상태, 상품이 1개 이상 등록된 상태)",
    "테스트단계": "1. 단계1\n2. 단계2\n3. 단계3",
    "기대결과": "정상 동작 시 나타나야 하는 결과",
    "우선순위": "High 또는 Medium 또는 Low"
  }
]

---

[티켓 정보]

티켓 키: {티켓 키 입력 예: MKQA-1}
티켓 유형: {Story / Task / Bug 중 선택}
티켓 제목: {티켓 제목 입력}

티켓 설명:
{티켓 설명 입력}
```

---

## 사용 예시

```
[티켓 정보]

티켓 키: MKQA-4
티켓 유형: Task
티켓 제목: 이메일 로그인 기능 구현

티켓 설명:
사용자가 이메일과 비밀번호를 입력하여 로그인할 수 있어야 한다.
- 이메일 형식이 아닌 경우 오류 메시지 표시
- 비밀번호 8자 미만 입력 시 오류 메시지 표시
- 5회 이상 실패 시 계정 잠금 처리
```

---

## 출력 결과 양식

| TC ID | 테스트 항목 | 사전 조건 | 테스트 단계 | 기대 결과 | 우선순위 |
|-------|-----------|---------|-----------|---------|---------|
| TC-001 | 정상 이메일/비밀번호로 로그인 | 가입된 계정이 존재하는 상태 | 1. 이메일 입력\n2. 비밀번호 입력\n3. 로그인 버튼 클릭 | 메인 화면으로 이동 | High |
| TC-002 | 잘못된 비밀번호 입력 시 오류 메시지 | 가입된 계정이 존재하는 상태 | 1. 이메일 입력\n2. 틀린 비밀번호 입력\n3. 로그인 버튼 클릭 | "비밀번호가 올바르지 않습니다" 메시지 표시 | High |
