## 2. watch_sheet.py — 구글 시트 폴링 기반 TC 자동 생성

### 동작 구조

```
구글 시트 '티켓 입력' 탭에 티켓 키/URL 입력
        ↓
GitHub Actions (10분마다 자동 실행)
        ↓
watch_sheet.py 실행
        ↓
B열이 비어있는 행 스캔 (미처리 감지)
        ↓
Jira 티켓 조회 → Groq AI TC 생성
        ↓
티켓 키 이름의 시트에 결과 기입 (예: MKQA-7 탭)
        ↓
B열 → '완료', C열 → 처리 시각 업데이트
        ↓
Slack 알림 발송
```

### 실행 방법

```bash
# 로컬 실행
python src/watch_sheet.py

# 특정 시트 ID 지정
python src/watch_sheet.py --sheet-id YOUR_SHEET_ID
```

### GitHub Actions 실행 방법

- **자동**: 구글 시트에 티켓 입력 후 최대 10분 대기
- **수동**: GitHub → Actions → CI → Run workflow 클릭

### 구글 시트 구조

**'티켓 입력' 탭 (입력)**

| A열 (티켓 URL 또는 이슈 키) | B열 (상태) | C열 (처리 시각) |
|---------------------------|-----------|----------------|
| MKQA-1 | 완료 | 2026-04-08 00:22:23 |
| MKQA-7 | 완료 | 2026-04-08 00:22:23 |
| MKQA-34 | ← 비워두면 대기 | |

**티켓별 탭 (출력, 자동 생성)**

각 티켓마다 별도 시트 생성 (예: MKQA-1, MKQA-7 탭)

| TC ID | 테스트 항목 | 테스트 환경 | 사전 조건 | 테스트 단계 | 기대 결과 | 실제 결과 | Pass/Fail | 우선순위 | 비고 |
|-------|-----------|-----------|---------|-----------|---------|---------|---------|---------|-----|
| TC-001 | ... | Web | ... | 1. ... | ... | (테스터 입력) | (테스터 입력) | High | |

## 3. QAT 중복 버그 티켓 수정

### 원인

Jira JQL에서 `summary ~ "[자동버그]"` 검색 시 대괄호 `[]`가 특수문자로 처리되어 항상 0건 반환
→ `existing_keys`가 항상 비어있어 매 실행마다 중복 티켓 생성

### 수정

```python
# 수정 전
summary ~ "[자동버그]"   # 대괄호 특수문자 → 0건

# 수정 후
summary ~ "자동버그"     # 정상 검색 → 기존 티켓 감지 → 중복 스킵
```

**파일**: `QATEST/utils/create_jira_bugs.py`
